# 需求规格说明书

## 1. 项目概述

### 1.1 项目背景
在多人脸素材采集中（如车载摄像头采集），如何快速高效地将多人脸素材清洗出来并自动标注人员位置，是一个亟待解决的问题。传统的人工标注方式效率低、成本高且容易出错。

### 1.2 项目目标
开发一套基于移动端的多人脸采集素材自动化清洗系统，实现：
1. 自动检测图片中的所有人脸
2. 识别人脸位置并分类（主驾/副驾/后排左/后排中/后排右）
3. 为同一个人分配唯一ID
4. 可视化标注显示结果
5. 移动端友好的交互界面

### 1.3 技术选型
| 层次 | 技术 |
|------|------|
| 后端服务 | FastAPI + Uvicorn |
| 人脸检测 | YOLOv8-face (ultralytics) |
| 人脸特征匹配 | InsightFace (ArcFace) |
| 座位分类 | 基于坐标的规则引擎 |
| 前端界面 | HTML5 + CSS3 + Vanilla JS（响应式移动端适配） |

## 2. 功能需求

### 2.1 核心功能

#### FR-01 人脸检测
- 上传图片后自动检测所有人脸
- 返回每个人脸的边界框坐标 `[x1, y1, x2, y2]`
- 返回检测置信度
- 支持同时检测5张以上人脸

#### FR-02 人员ID分配
- 基于人脸特征向量（512维）进行跨图片匹配
- 使用余弦相似度计算人脸相似度
- 同一个人在不同图片中保持唯一且固定的ID
- 相似度阈值可配置（默认0.5）

#### FR-03 座位位置分类
- 根据人脸框中心坐标映射到座位区域
- 5个座位区域：主驾、副驾、后排左、后排中、后排右
- 座位区域配置可通过API动态调整

#### FR-04 可视化标注
- 在原始图片上绘制人脸框
- 标注座位名称和人员ID
- 使用不同颜色区分不同座位区域

#### FR-05 单张检测模式
- 上传单张图片，即时返回检测结果
- 显示处理耗时

#### FR-06 批量清洗模式
- 支持批量上传多张图片
- 跨图片统一人员ID
- 生成统计报告（每人出现次数、主要座位）

### 2.2 非功能需求

#### NFR-01 性能
- 单张图片检测响应时间 < 5秒（CPU环境）
- 支持并发请求处理

#### NFR-02 兼容性
- 支持 JPG、PNG 格式图片
- 前端适配移动端（响应式设计）

#### NFR-03 易用性
- 拖拽上传图片
- 一键切换单张/批量模式
- 结果直观展示（标注图+列表）

## 4. 接口设计

### 4.1 POST /api/detect - 单张图片检测

**请求：**
- Content-Type: multipart/form-data
- 参数: `file` (图片文件)

**响应：**
```json
{
  "image_id": 0,
  "filename": "car_01.jpg",
  "num_faces": 3,
  "faces": [
    {
      "bbox": [120, 80, 200, 170],
      "conf": 0.92,
      "seat": "master_driver",
      "seat_label": "主驾",
      "person_id": 1
    }
  ],
  "annotated_image_base64": "data:image/jpeg;base64,...",
  "processing_time_ms": 2340.5
}
```

### 4.2 POST /api/detect-batch - 批量检测

**请求：**
- Content-Type: multipart/form-data
- 参数: `files` (多张图片文件)

**响应：**
```json
{
  "total_images": 5,
  "total_faces": 12,
  "processing_time_ms": 15600.0,
  "results": [...]
}
```

### 4.3 GET /api/seat-zones - 座位区域配置

**响应：**
```json
{
  "zones": {
    "master_driver": {"x_range": [0.5, 1.0], "y_range": [0.5, 1.0]},
    ...
  }
}
```

## 5. 座位分类规则

基于图像坐标归一化（0-1范围）：

| 座位ID | X范围 | Y范围 | 颜色 |
|--------|-------|-------|------|
| posterior_left | 0.00-0.33 | 0.00-0.50 | 红色 |
| posterior_right | 0.33-0.66 | 0.00-0.50 | 绿色 |
| after_the_middle | 0.66-1.00 | 0.00-0.50 | 蓝色 |
| co_driver | 0.00-0.50 | 0.50-1.00 | 橙色 |
| master_driver | 0.50-1.00 | 0.50-1.00 | 青色 |

注：Y轴上方为后排，下方为前排；X轴左侧为左，右侧为右。
