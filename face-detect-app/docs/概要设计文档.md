## 2. 核心模块详细设计

### 2.1 FaceDetector（人脸检测模块）

**职责**：使用YOLOv8-face检测图片中所有人脸

```python
class FaceDetector:
    def detect(image) -> list[dict]:
        """返回 [{bbox: [x1,y1,x2,y2], conf: float}, ...]"""

    # 流程：
    # 1. 加载预训练模型 yolov8n-face.pt
    # 2. 调用 model.predict() 进行推理
    # 3. 解析输出，提取bbox和置信度
    # 4. 按x坐标排序（左→右）
```

**关键参数**：
- 置信度阈值：`conf=0.5`（可调整）
- 模型：`yolov8n-face.pt`（轻量，适合移动端模拟）
- 设备：CPU（可通过`device="cuda"`启用GPU加速）

### 2.2 FaceRecognizer（人脸特征匹配模块）

**职责**：提取人脸特征并跨图片分配统一ID

```python
class FaceRecognizer:
    def extract_embedding(image, bbox) -> np.ndarray:
        """提取512维人脸特征向量"""

    def match_faces_across_images(faces, threshold=0.5):
        """跨图片匹配，分配person_id"""
        # 策略：
        # 1. 逐图片处理，维护person_db
        # 2. 对新检测到的人脸，与person_db中所有人对比
        # 3. 最大相似度 >= threshold → 归为同一人
        # 4. 否则分配新person_id
```

**匹配策略**：
- 特征向量：512维归一化向量（InsightFace ArcFace）
- 相似度度量：余弦相似度 `sim(a,b) = dot(a,b) / (||a||·||b||)`
- 阈值：0.5（可调，0.3-0.7之间测试最佳值）

### 2.3 SeatClassifier（座位分类模块）

**职责**：基于人脸框坐标判断座位位置

```python
class SeatClassifier:
    def classify(bbox, img_w, img_h) -> str:
        """
        计算人脸框中心点 (cx, cy)，归一化到 0-1
        遍历SEAT_ZONES匹配区域
        """
        cx = (bbox[0] + bbox[2]) / 2 / img_w
        cy = (bbox[1] + bbox[3]) / 2 / img_h
        # 依次检查5个区域
```

**区域定义**：
```
   Y: 0.0 ─────────────────── 1.0
        │  后排(0.0-0.5)  │ 前排(0.5-1.0) │
   X:   │ 左  │  中   │  右  │ 左  │  右   │
  0.0   │     │       │     │     │       │
  0.33  │ 后排左      后排中     │ 副驾       │
  0.66  │            后排右     │    主驾    │
  1.0   │                     │           │
```

### 2.4 FaceAnnotator（可视化标注模块）

**职责**：在图像上绘制检测结果

```
绘制内容：
1. 人脸矩形框（颜色对应座位区域）
2. 标签背景 + 文字（座位名称 + ID）
3. 置信度显示
```

## 3. 数据流

### 3.1 单张图片检测流程

```
用户上传图片
    │
    ▼
[HTTP POST /api/detect]
    │
    ▼
解码图片 (cv2.imdecode)
    │
    ▼
FaceDetector.detect(image)
    │ → YOLOv8-face 推理
    │ → [{bbox, conf}, ...]
    ▼
FaceRecognizer.extract_embedding(image, bbox)  [对每个人脸]
    │ → InsightFace 特征提取
    │ → 512维向量
    ▼
FaceRecognizer.match_faces_across_images()
    │ → 分配 person_id
    ▼
SeatClassifier.classify(bbox, w, h)  [对每个人脸]
    │ → 座位分类
    ▼
FaceAnnotator.draw(image, faces)
    │ → OpenCV绘制
    ▼
返回 JSON + 标注图(base64)
```

### 3.2 批量图片清洗流程

```
批量上传N张图片
    │
    ▼
依次调用FaceDetector + extract_embedding
    │ → 收集全部人脸（含image_id标记）
    ▼
一次 match_faces_across_images（全量匹配）
    │ → 跨图统一person_id
    ▼
逐图 classify + annotate
    │
    ▼
返回汇总统计 + 每张图标注结果
```

## 4. API 详细设计

### 4.1 路由清单

| 方法 | 路径 | 功能 | 响应格式 |
|------|------|------|----------|
| GET | `/` | 前端页面 | HTML |
| GET | `/api/health` | 健康检查 | JSON |
| POST | `/api/detect` | 单张图片检测 | JSON |
| POST | `/api/detect-batch` | 批量图片检测 | JSON |
| GET | `/api/seat-zones` | 获取座位配置 | JSON |

### 4.2 错误处理

| HTTP状态码 | 场景 |
|-----------|------|
| 400 | 图片格式不支持或无法解析 |
| 413 | 文件过大 |
| 500 | 模型推理异常 |

## 5. 前端设计

### 5.1 页面结构

```
Header（标题栏）
├── 座位图例卡片
├── 模式切换标签（单张/批量）
├── 上传区域（拖拽 + 点击）
├── 结果展示区域
│   ├── 单张模式：检测图 + 人脸列表
│   └── 批量模式：统计表 + 逐图结果
└── Loading 遮罩
```

### 5.2 移动端适配

- viewport 设置：`width=device-width, maximum-scale=1.0`
- 卡片圆角 + 阴影
- 触摸友好的按钮尺寸（最小 44px）
- 网格布局自适应（480px断点）

### 5.3 交互流程

```
单张模式：
  上传 → Loading → 展示标注图 + 人脸列表

批量模式：
  多选上传 → Loading → 统计表格 → 逐图展开
```
