# 课题2后续完成说明（不依赖额外数据集）

## 项目目标

本项目对应“多人脸采集素材自动化清洗”。系统上传一张或多张车内多人脸图片后，输出每个人的人脸框坐标、人员 ID 和座位位置，并生成带中文标签的标注图。座位类别固定为：

- `master_driver`：主驾
- `co_driver`：副驾
- `posterior_left`：后排左
- `posterior_right`：后排中
- `after_the_middle`：后排右

## 当前技术路线

当前版本不下载外部数据集，不重新训练模型。系统采用：

- YOLOv8-face / Haar Cascade fallback：检测人脸框坐标和置信度。
- InsightFace / 像素特征 fallback：在批量图片中尽量匹配同一人员 ID。
- 固定视角规则分类器：基于人脸框位置、宽度占比、面积占比和多人相对位置判断座位。
- FastAPI + HTML 前端：提供单张检测、批量检测、标注图展示和结果汇总。

## 适用范围与局限性

当前座位分类属于规则法，主要适用于固定或近似固定车内摄像头视角。它可以完成课程项目的端到端演示，但不能等同于工业级 OMS/DMS 乘员监测系统。

适用场景：

- 同一类车内广角视角。
- 前排双人、后排三人、五座满员等常规测试图。
- AI 生成图或项目样例图中的基础检测与标注。

局限场景：

- 摄像头安装位置变化较大。
- 车型、焦距、透视关系明显变化。
- 大角度侧脸、严重遮挡、暗光、强光。
- 仅靠人脸框无法区分前后排的近似构图。

基础验收：

- 系统可以启动并通过 HTTP 页面访问。
- 单张图片检测接口正常。
- 批量图片检测接口正常。
- 标注图可以显示人脸框、座位名和人员 ID。

样例验收：

- 至少 5 张单图样例。
- 至少 1 组批量检测样例。
- 每类座位至少有一个可展示成功样例。
- `acceptance_test_records.csv` 记录预期人数、实际人数、预期座位、实际座位、结果和问题说明。
