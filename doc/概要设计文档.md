**关键参数**：
- 置信度阈值：`conf=0.1`（Pipeline 初始化默认值，可调）
- 候选模型：`yolov8n-face-lindevs.pt`（优先）→ `yolov8n-face.pt`（备选）

**匹配策略**：

- 默认 `use_recognition=False`，此时 person_id 为每图内顺序编号

### 2.3 SeatClassifier（座位分类模块）

**职责**：基于人脸框坐标 + 自适应前后排判断，分类座位位置

```python
        # 1. 自适应判断前后排（_is_front_row）
        # 2. 前排：nx < 0.50 → co_driver, nx >= 0.50 → master_driver
        # 3. 后排：nx < 0.44 → posterior_left
        #        0.44 <= nx < 0.56 → posterior_right
        #        nx >= 0.56 → after_the_middle
```

**前后排判断逻辑**（`_is_front_row`）：
- 单脸：综合判断——人脸框宽度/面积足够大（> 8% / > 0.010），且位置靠边（|X-0.5|≥0.18）或靠下（Y≥0.28）
- 双脸：Y 坐标跨度 > 0.12 或平均宽度 > 0.08 → 前排
- 3+ 人脸：优先调用 `_front_candidates_by_position()` 多维评分（X偏移×0.35 + 宽度×0.45 + 面积×0.20 + Y位置×0.12），两侧位置+尺寸足够大才判定前排；若该判定失败，退化为 2-means 聚类（weighted [height, 3×Y-center]），高均值簇为前排，辅以 tallest-in-back 翻转、Y偏离修正、高度比>2.0 修正

**区域参考定义**：

| 座位ID | X范围 | Y范围 | 颜色 | 含义 |
|--------|-------|-------|------|------|
| `master_driver` | 0.60-1.00 | 0.30-0.55 | 青色 | 前排右手边 |
| `co_driver` | 0.00-0.40 | 0.30-0.55 | 橙色 | 前排左手边 |
| `posterior_left` | 0.25-0.45 | 0.25-0.45 | 红色 | 后排左侧 |
| `posterior_right` | 0.45-0.65 | 0.25-0.45 | 绿色 | 后排中间 |
| `after_the_middle` | 0.65-0.90 | 0.25-0.50 | 蓝色 | 后排右侧 |

注：SEAT_ZONES 仅作为参考区域，实际分类通过自适应前后排判断 + X 坐标细分完成。

## 3. 数据流

### 3.1 单张图片检测流程

```
用户上传图片
    │
    ▼
[HTTP POST /api/detect]
    │
    ▼
解码图片 (cv2.imdecode)
    │
    ▼
Pipeline.process_image(image, image_id=0)
    │
    ▼
FaceDetector.detect(image)
    │ → YOLOv8-face 推理（或 Haar Cascade fallback）
    │ → [{bbox, conf}, ...]
    ▼
（可选）FaceRecognizer.extract_embedding(image, bbox)
    │ → InsightFace 512维向量 或 像素直方图256维
    ▼
（可选）FaceRecognizer.match_faces_across_images()
    │ → 分配 person_id（跨图匹配）
    │ 默认模式：person_id = 顺序编号
    ▼
SeatClassifier.classify(bbox, w, h, all_bboxes)
    │ → 自适应前后排判断
    │ → X 坐标细分座位
    ▼
FaceAnnotator.draw(image, faces)
    │ → PIL 绘制中文标注
    ▼
保存标注图 (data/results/) + 编码为 base64
    │
    ▼
返回 JSON {image_id, filename, num_faces, faces[], annotated_image_base64, processing_time_ms}
```

### 3.2 批量图片清洗流程

```
批量上传 N 张图片
    │
    ▼
[HTTP POST /api/detect-batch]
    │
    ▼
Pipeline.process_batch(images)
    │
    ▼
对每张图：FaceDetector.detect(image)
    │ → 收集全部人脸（含 image_id 标记）
    ▼
（可选）提取特征 + 跨全量图片匹配
    │ → match_faces_across_images(all_faces)
    ▼
重新按 image_id 分组
    ▼
逐图：SeatClassifier.classify() + FaceAnnotator.draw()
    │
    ▼
保存结果图 (data/results/batch_*.jpg)
    │
    ▼
返回汇总统计 + 每张图标注结果
{
  "total_images": N,
  "total_faces": M,
  "processing_time_ms": T,
  "results": [{image_id, filename, num_faces, faces[], annotated_image_base64}, ...]
}
```

## 4. API 详细设计

### 4.1 路由清单

| 方法 | 路径 | 功能 | 响应格式 |
|------|------|------|----------|
| GET | `/` | 前端页面 | HTML |
| GET | `/setup` | 配网页面 | HTML |
| GET | `/api/health` | 健康检查 | JSON |
| GET | `/api/seat-zones` | 座位区域配置 | JSON |
| GET | `/api/server-info` | 服务器信息 | JSON |
| POST | `/api/detect` | 单张图片检测 | JSON |
| POST | `/api/detect-batch` | 批量图片检测 | JSON |

### 4.2 请求/响应详情

#### POST /api/detect

- Content-Type: multipart/form-data
- 参数: `file` (UploadFile)
- 保存原始图到 `data/uploads/`，标注图到 `data/results/`

#### POST /api/detect-batch

- Content-Type: multipart/form-data
- 参数: `files` (多张 UploadFile)
- 跳过无法解析的图片，返回有效图片结果

### 4.3 错误处理

| HTTP状态码 | 场景 |
|-----------|------|
| 400 | 图片格式不支持或无法解析 / 未找到有效图片 |
| 413 | 文件过大（FastAPI 默认限制） |
| 500 | 模型推理异常（自动打印 traceback） |

## 5. 前端设计

### 5.1 页面结构

```
Header（标题栏）
├── 座位图例卡片（5色图例）
├── 模式切换标签（单张检测 / 批量清洗）
├── 上传区域（拖拽 + 点击选择文件）
├── 结果展示区域
│   ├── 单张模式：标注图 + 人脸列表（含座位标签、置信度条）
│   └── 批量模式：统计表 + 逐图展开结果
└── Loading 遮罩层
```

### 5.2 配网页面（/setup）

```
Header（服务器信息）
├── 二维码（qrcode API 生成，含服务器LAN URL）
├── 手动输入 URL + 连接验证按钮
├── 连接状态提示
└── localStorage 保存服务器地址（供 Android WebView 读取）
```

### 5.3 移动端适配

- viewport 设置：`width=device-width, initial-scale=1.0`
- 响应式断点：480px（移动端 < 480px，桌面端 >= 480px）
- 卡片圆角 + 阴影
- 触摸友好的按钮尺寸
- 拖拽上传 + 文件选择器双模式

### 5.4 交互流程

```
单张模式：
  上传 → Loading 遮罩 → 展示标注图 + 人脸列表（座位标签 + 置信度）

批量模式：
  多选上传 → Loading 遮罩 → 统计表格 → 逐图展开标注结果

配网流程：
  打开 /setup → 扫码 / 手动输入 → 验证连接 → 返回主页
```

### 5.5 Android 客户端

- **入口**: `MainActivity` — WebView 加载前端页面
- **桥接**: `JsBridge` — JavaScript ↔ Java 通信
- **配网**: `SetupActivity` — 扫描二维码获取服务器地址
- **设置**: `SettingsActivity` — 服务器地址管理
- **启动页**: `SplashActivity` — 闪屏页
- 网络配置：允许 cleartext HTTP（Android Network Security Config）
- 支持 Android 8.0 (API 26) ~ Android 14 (API 34)
