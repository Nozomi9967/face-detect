# 需求规格说明书

## 1. 项目概述

### 1.1 项目背景
在多人脸素材采集中（如车载摄像头采集），如何快速高效地将多人脸素材清洗出来并自动标注人员位置，是一个亟待解决的问题。传统的人工标注方式效率低、成本高且容易出错。

### 1.2 项目目标
开发一套基于移动端的多人脸采集素材自动化清洗系统，实现：
1. 自动检测图片中的所有人脸
2. 识别人脸位置并分类（主驾/副驾/后排左/后排中/后排右）
3. 为同一个人分配唯一ID
4. 可视化标注显示结果
5. 移动端友好的交互界面

### 1.3 技术选型
| 层次 | 技术 |
|------|------|
| 后端服务 | FastAPI + Uvicorn |
| 人脸检测 | YOLOv8-face (ultralytics) |
| 人脸特征匹配 | InsightFace (ArcFace) |
| 座位分类 | 基于坐标的规则引擎 |
| 前端界面 | HTML5 + CSS3 + Vanilla JS（响应式移动端适配） |

## 2. 功能需求

### 2.1 核心功能

#### FR-01 人脸检测
- 上传图片后自动检测所有人脸
- 返回每个人脸的边界框坐标 `[x1, y1, x2, y2]`
- 返回检测置信度
- 支持同时检测5张以上人脸
- 检测后端：YOLOv8-face（优先）/ OpenCV Haar Cascade（fallback）
- 置信度阈值默认 `0.1`（可通过初始化参数调整）

#### FR-02 人员ID分配
- 默认模式：每张图内人脸分配顺序编号（1, 2, 3, ...）
- 可选模式（需启用 `use_recognition=True`）：基于 InsightFace ArcFace 512维特征向量进行跨图片匹配
- 同一个人在不同图片中保持唯一且固定的ID（仅在启用特征匹配时）
- 相似度阈值可配置（默认0.5）

#### FR-03 座位位置分类
- 根据人脸框中心坐标映射到座位区域
- 5个座位区域：主驾、副驾、后排左、后排中、后排右
- 自适应前后排判断算法（多维评分 + 2-means 聚类）
- 座位区域配置可通过API动态获取

#### FR-04 可视化标注
- 在原始图片上绘制人脸框
- 标注格式：`座位名称 ID:N`（如 "主驾 ID:1"）
- 使用 PIL + 中文字体绘制标注（支持中文显示）
- 不同座位使用不同颜色区分
- 同时显示置信度

#### FR-05 单张检测模式
- 上传单张图片，即时返回检测结果
- 显示处理耗时（毫秒级）
- 展示标注图 + 人脸列表

#### FR-06 批量清洗模式
- 支持批量上传多张图片
- 跨图片统一人员ID（仅在启用特征匹配时）
- 返回汇总统计（总图片数、总人脸数） + 每张图片的检测结果及标注图

#### FR-07 配网与移动端
- `/setup` 页面展示服务器二维码，手机扫码获取局域网地址
- `/api/server-info` 提供服务器URL
- 支持 Android WebView 原生客户端（`android/` 目录）

### 2.2 非功能需求

#### NFR-01 性能
- 单张图片检测响应时间 < 5秒（CPU环境）
- 支持并发请求处理

#### NFR-02 兼容性
- 支持 JPG、PNG 格式图片
- 前端适配移动端（响应式设计）

#### NFR-03 易用性
- 拖拽上传图片
- 一键切换单张/批量模式
- 结果直观展示（标注图+列表）

## 5. 座位分类规则

座位分类采用自适应算法，根据人脸框的位置和大小自动判断前后排，再按 X 坐标映射到具体座位。SEAT_ZONES 定义了参考区域：

| 座位ID | X范围 | Y范围 | 颜色 | 含义 |
|--------|-------|-------|------|------|
| posterior_left | 0.25-0.45 | 0.25-0.45 | 红色 | 后排左侧 |
| posterior_right | 0.45-0.65 | 0.25-0.45 | 绿色 | 后排中间 |
| after_the_middle | 0.65-0.90 | 0.25-0.50 | 蓝色 | 后排右侧 |
| co_driver | 0.00-0.40 | 0.30-0.55 | 橙色 | 前排左手边 |
| master_driver | 0.60-1.00 | 0.30-0.55 | 青色 | 前排右手边 |

**分类逻辑**（`seat_classifier.py`）：
1. 计算所有人脸框中心归一化坐标（X, Y）、宽度、面积
2. 单脸：综合判断——人脸框宽度/面积足够大，且位置靠边或靠下
3. 双脸：比较 Y 坐标跨度或平均宽度判断
4. 3+ 人脸：优先多维评分（X偏移×0.35 + 宽度×0.45 + 面积×0.20 + Y位置×0.12），两侧位置+尺寸足够大才判定前排；若该判定失败，退化为 2-means 聚类（weighted [height, 3×Y-center]），高均值簇为前排，并辅以 tallest-in-back 翻转、Y偏离修正、高度比>2.0 修正
5. 前排根据 X 坐标区分：`X < 0.50` 为副驾，`X >= 0.50` 为主驾
6. 后排根据 X 坐标三分：`X < 0.44` 后排左，`0.44 <= X < 0.56` 后排中，`X >= 0.56` 后排右

注：实际分类不直接依赖 SEAT_ZONES 的坐标范围进行匹配，而是先通过自适应算法确定前后排，再用 X 坐标细分。

## 6. 标注显示规范

- 标注标签格式：`座位名称 ID:N`（如 `主驾 ID:1`）
- 使用 PIL + 中文字体（simhei.ttf / msyh.ttc）绘制中文标签
- 不同座位使用不同颜色区分
- 标注图通过 base64 编码返回前端展示
- 置信度值显示在标注框下方
