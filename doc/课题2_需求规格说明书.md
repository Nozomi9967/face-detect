# 课题2：多人脸采集素材自动化清洗 — 需求规格说明书

## 1. 项目背景

在多人脸素材采集场景中（如车载摄像头采集），每个被采集对象轮流在不同的位置上做相应动作。传统人工标注方式效率低、成本高且容易出错。如何快速高效地将多人脸采集素材清洗出来，同时方便后续自动打标，成为一个亟待解决的问题。

## 2. 项目目标

开发一个移动端程序（APP / 小程序均可），上传包含多人脸的图片后，能够高效准确地完成以下任务：

1. **人脸检测** — 找到图中所有人脸及其边界框坐标
2. **ID 分配** — 为每个人分配唯一且固定的 ID
3. **位置分类** — 识别人脸在车内所处的座位位置
4. **可视化标注** — 标注显示人脸框、ID 和位置名称
5. **极端 case 处理** — 支持大角度、遮挡较多、暗光、强光等场景

## 3. 功能需求

### 3.1 核心功能

#### FR-01 人脸检测
- 上传图片后自动检测图中所有人脸
- 返回每个人脸的边界框坐标 `[x1, y1, x2, y2]`
- 返回检测置信度
- 支持同时检测 5 张以上人脸
- 检测后端：YOLOv8-face（优先）/ OpenCV Haar Cascade（fallback）
- 置信度阈值默认 `0.1`（可调）

#### FR-02 人员 ID 分配
- 默认模式：每张图内人脸分配顺序编号（1, 2, 3, ...）
- 可选模式：基于 InsightFace ArcFace 512维特征向量进行跨图片匹配（需启用）
- 同一个人在不同图片中保持唯一且固定的 ID（仅在启用特征匹配时）
- ID 分配结果需在标注图中清晰显示

#### FR-03 座位位置分类
- 根据人脸框坐标映射到车内座位区域
- 5 个座位区域定义：

| 座位 ID | 名称 | 含义 |
|---------|------|------|
| `master_driver` | 主驾 | 前排右手边 |
| `co_driver` | 副驾 | 前排左手边 |
| `posterior_left` | 后排左 | 后排左侧 |
| `posterior_right` | 后排中 | 后排中间 |
| `after_the_middle` | 后排右 | 后排右侧 |

- 自适应前后排判断（多维评分 + 2-means 聚类，基于位置/尺寸/面积/高度）

#### FR-04 可视化标注
- 在原始图片上绘制人脸框
- 标注格式：`座位名称 ID:N`（如 "主驾 ID:1"）
- 使用 PIL + 中文字体绘制（支持中文标签）
- 使用不同颜色区分不同座位区域

#### FR-05 批量清洗模式
- 支持批量上传多张图片
- 汇总统计（总图片数、总人脸数）
- 逐图展示检测结果及标注图

#### FR-06 配网与移动端
- 配网页面（`/setup`）展示服务器二维码
- 手机扫码获取局域网服务器地址
- 支持 Android WebView 原生客户端

### 3.2 非功能需求

#### NFR-01 性能
- 单张图片检测响应时间 < 5 秒
- 支持并发请求处理

#### NFR-02 兼容性
- 支持 JPG、PNG 格式图片
- 移动端界面友好

## 6. 座位分类规则

座位分类采用自适应算法，根据人脸框的位置和大小自动判断前后排，再按 X 坐标映射到具体座位。SEAT_ZONES 定义了参考区域：

| 座位ID | X范围 | Y范围 | 含义 |
|---------|------|-------|------|
| `master_driver` | 0.60-1.00 | 0.30-0.55 | 前排右手边 |
| `co_driver` | 0.00-0.40 | 0.30-0.55 | 前排左手边 |
| `posterior_left` | 0.25-0.45 | 0.25-0.45 | 后排左侧 |
| `posterior_right` | 0.45-0.65 | 0.25-0.45 | 后排中间 |
| `after_the_middle` | 0.65-0.90 | 0.25-0.50 | 后排右侧 |

**分类逻辑**（`seat_classifier.py`）：
1. 计算所有人脸框中心归一化坐标（X, Y）、宽度、面积
2. 单脸：综合判断——人脸框宽度/面积足够大，且位置靠边或靠下
3. 双脸：比较 Y 坐标跨度或平均宽度判断
4. 3+ 人脸：优先多维评分（X偏移×0.35 + 宽度×0.45 + 面积×0.20 + Y位置×0.12），两侧位置+尺寸足够大才判定前排；若该判定失败，退化为 2-means 聚类（weighted [height, 3×Y-center]），高均值簇为前排，辅以 tallest-in-back 翻转、Y偏离修正、高度比>2.0 修正
5. 前排根据 X 坐标区分：`X < 0.50` 为副驾，`X >= 0.50` 为主驾
6. 后排根据 X 坐标三分：`X < 0.44` 后排左，`0.44 <= X < 0.56` 后排中，`X >= 0.56` 后排右

注：实际分类不直接依赖 SEAT_ZONES 的坐标范围进行匹配，而是先通过自适应算法确定前后排，再用 X 坐标细分。

## 7. 标注显示规范

- 标注标签格式：`座位名称 ID:N`（如 `主驾 ID:1`）
- 使用 PIL + 中文字体（simhei.ttf / msyh.ttc）绘制中文标签
- 不同座位使用不同颜色区分
- 标注图以 base64 格式返回前端展示
- 置信度值显示在标注框下方

## 8. 评分要点

| 评分维度 | 权重 | 要点 |
|---------|------|------|
| 人脸检测准确性 | 高 | 多角度、遮挡、光照下的检测率 |
| ID 分配正确性 | 高 | 同人同 ID、不同人不同 ID |
| 座位分类准确性 | 中 | 5 个座位区域映射准确 |
| 极端 case 处理 | 中 | 大角度、遮挡、暗光、强光 |
| 移动端体验 | 低 | 界面美观、操作流畅 |
| 代码质量 | 低 | 注释清晰、结构合理 |
