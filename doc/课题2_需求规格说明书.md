# 课题2：多人脸采集素材自动化清洗 — 需求规格说明书

## 1. 项目背景

在多人脸素材采集场景中（如车载摄像头采集），每个被采集对象轮流在不同的位置上做相应动作。传统人工标注方式效率低、成本高且容易出错。如何快速高效地将多人脸采集素材清洗出来，同时方便后续自动打标，成为一个亟待解决的问题。

## 2. 项目目标

开发一个移动端程序（APP / 小程序均可），上传包含多人脸的图片后，能够高效准确地完成以下任务：

1. **人脸检测** — 找到图中所有人脸及其边界框坐标
2. **ID 分配** — 为每个人分配唯一且固定的 ID
3. **位置分类** — 识别人脸在车内所处的座位位置
4. **可视化标注** — 标注显示人脸框、ID 和位置名称
5. **极端 case 处理** — 支持大角度、遮挡较多、暗光、强光等场景

## 3. 功能需求

### 3.1 核心功能

#### FR-01 人脸检测
- 上传图片后自动检测图中所有人脸
- 返回每个人脸的边界框坐标 `[x1, y1, x2, y2]`
- 返回检测置信度
- 支持同时检测 5 张以上人脸

#### FR-02 人员 ID 分配
- 基于人脸特征进行跨图片匹配
- 同一个人在不同图片中保持唯一且固定的 ID
- ID 分配结果需在标注图中清晰显示

#### FR-03 座位位置分类
- 根据人脸框坐标映射到车内座位区域
- 5 个座位区域定义：

| 座位 ID | 名称 | 含义 |
|---------|------|------|
| `master_driver` | 主驾 | 前排右手边 |
| `co_driver` | 副驾 | 前排左手边 |
| `posterior_left` | 后排左 | 后排左侧 |
| `posterior_right` | 后排中 | 后排中间 |
| `after_the_middle` | 后排右 | 后排右侧 |

#### FR-04 可视化标注
- 在原始图片上绘制人脸框
- 标注座位名称和人员 ID
- 使用不同颜色区分不同座位区域

#### FR-05 极端 case 处理
- 大角度偏转人脸检测
- 遮挡较多的人脸识别
- 暗光 / 强光条件下的检测

### 3.2 非功能需求

#### NFR-01 性能
- 单张图片检测响应时间 < 5 秒
- 支持并发请求处理

#### NFR-02 兼容性
- 支持 JPG、PNG 格式图片
- 移动端界面友好

## 5. API 接口设计

### 5.1 POST /api/detect — 单张图片检测

**请求：**
- Content-Type: multipart/form-data
- 参数: `file` (图片文件)

**响应：**
```json
{
  "filename": "car_01.jpg",
  "num_faces": 3,
  "faces": [
    {
      "bbox": [120, 80, 200, 170],
      "conf": 0.92,
      "seat": "master_driver",
      "seat_label": "主驾",
      "person_id": 1
    },
    {
      "bbox": [50, 100, 180, 240],
      "conf": 0.87,
      "seat": "posterior_left",
      "seat_label": "后排左",
      "person_id": 2
    }
  ],
  "annotated_image_base64": "data:image/jpeg;base64,...",
  "processing_time_ms": 2340.5
}
```

### 5.2 GET /api/seat-zones — 座位区域配置

**响应：**
```json
{
  "zones": {
    "master_driver": {"x_range": [0.50, 1.00], "y_range": [0.50, 1.00]},
    "co_driver":     {"x_range": [0.00, 0.50], "y_range": [0.50, 1.00]},
    "posterior_left":  {"x_range": [0.00, 0.33], "y_range": [0.00, 0.50]},
    "posterior_right": {"x_range": [0.33, 0.66], "y_range": [0.00, 0.50]},
    "after_the_middle":{"x_range": [0.66, 1.00], "y_range": [0.00, 0.50]}
  }
}
```

## 6. 座位坐标映射规则

基于图像坐标归一化（0~1 范围），Y 轴上方为后排，下方为前排：

```
   Y: 0.0 ───────────────────────────── 1.0
        │         后排 (0.0-0.50)        │  前排 (0.50-1.00)  │
   X:   │    左     │      中     │    右 │     左    │    右    │
  0.0   │           │             │       │          │          │
  0.33  │ posterior  │ posterior    │       │  co_     │ master_  │
        │   _left    │   _right     │       │  driver  │  driver  │
  0.66  │  (后排左)   │  (后排中)    │ after │ (副驾)    │  (主驾)   │
        │           │             │ _the_ │          │          │
  1.0   │           │             │ middle│          │          │
        │           │             │(后排右)│          │          │
```

**注**：人脸框中心点 `(cx, cy)` 归一化后按上述区域匹配。

## 7. 标注显示规范

- 同一个人 ID 固定且唯一
- 每个人脸框上方标注：`座位名称 + ID`（例：`主驾 #1`）
- 不同座位使用不同颜色区分
- 标注图以 base64 格式返回前端展示

## 8. 评分要点

| 评分维度 | 权重 | 要点 |
|---------|------|------|
| 人脸检测准确性 | 高 | 多角度、遮挡、光照下的检测率 |
| ID 分配正确性 | 高 | 同人同 ID、不同人不同 ID |
| 座位分类准确性 | 中 | 5 个座位区域映射准确 |
| 极端 case 处理 | 中 | 大角度、遮挡、暗光、强光 |
| 移动端体验 | 低 | 界面美观、操作流畅 |
| 代码质量 | 低 | 注释清晰、结构合理 |
